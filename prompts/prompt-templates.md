# Prompt Templates

AI 도구와 효과적으로 협업하기 위한 재사용 가능한 프롬프트 모음입니다.

## 1. 초기 프로젝트 설정

```markdown
나는 Incident Analysis System을 구현하려고 합니다.

다음 스펙 문서들을 읽고 프로젝트를 시작해주세요:
- specs/requirements.md
- specs/architecture.md
- specs/api-spec.md
- specs/scenarios/

구현은 다음 순서로 진행합니다:
1. 프로젝트 구조 생성 (Spring Boot, Gradle)
2. 인프라 설정 (Docker Compose - Kafka, MySQL, Redis)
3. Detection Engine 구현
4. AI Analysis 통합
5. Web UI 개발

첫 단계로 프로젝트 구조를 생성해주세요.
```

## 2. 스펙 전달 (컨텍스트 제공)

```markdown
[specs/{filename}.md의 내용을 복사하여 붙여넣기]

위 스펙을 기반으로 {컴포넌트명}을 구현해주세요.
다음 사항을 준수해주세요:
- Java 17, Spring Boot 3.x 사용
- 테스트 코드 포함
- 적절한 예외 처리
- 로깅 추가
```

## 3. 구현 요청

```markdown
{컴포넌트명}을 구현해주세요.

요구사항:
- [구체적인 기능 목록]

제약사항:
- [기술 스택, 라이브러리 제한]

참고:
- specs/architecture.md의 {섹션} 참조
```

## 4. 디버깅 요청

```markdown
다음 오류가 발생했습니다:

[에러 로그 복사]

원인을 파악하고 수정해주세요.

관련 파일:
- {file1.java}
- {file2.java}
```

## 5. 코드 리뷰 요청

```markdown
다음 코드를 리뷰해주세요:

[코드 붙여넣기]

다음 관점에서 검토해주세요:
- 코드 품질 (가독성, 유지보수성)
- 성능 이슈
- 보안 취약점
- 베스트 프랙티스 준수 여부
```

## 6. 테스트 작성 요청

```markdown
{클래스명}에 대한 테스트 코드를 작성해주세요.

커버해야 할 시나리오:
- [정상 케이스]
- [예외 케이스]
- [경계값 테스트]

JUnit 5와 Mockito를 사용해주세요.
```

## 7. 문서화 요청

```markdown
{컴포넌트/API}에 대한 문서를 작성해주세요.

포함 내용:
- 개요 및 목적
- 주요 기능
- API 엔드포인트 (Request/Response 예시)
- 사용 예제
- 주의사항
```

## 8. 리팩토링 요청

```markdown
다음 코드를 리팩토링해주세요:

[코드 붙여넣기]

목표:
- 코드 중복 제거
- 메서드 분리 (Single Responsibility)
- 네이밍 개선
- 불필요한 복잡도 제거

단, 기능은 동일하게 유지해주세요.
```

## 효과적인 프롬프트 작성 팁

### ✅ Do
- 명확하고 구체적인 요구사항 제시
- 관련 스펙 문서 또는 코드 컨텍스트 제공
- 제약사항 및 기술 스택 명시
- 예상 결과물 설명
- 단계별로 나누어 요청

### ❌ Don't
- 모호하거나 추상적인 요청
- 너무 많은 작업을 한 번에 요청
- 컨텍스트 없이 "만들어줘" 식 요청
- 스펙과 모순되는 요구사항
- AI의 이전 응답을 무시한 반복 요청

## 프롬프트 패턴별 효과 측정

각 구현 후 `prompts/{implementation}.md`에서 다음 분석:
- 어떤 프롬프트 패턴이 효과적이었나?
- AI가 한 번에 이해한 요청 vs 재설명 필요 요청
- 명확한 결과를 얻기 위한 최적 프롬프트 길이
- 컨텍스트 제공 방법 (파일 전체 vs 요약 vs 참조)
